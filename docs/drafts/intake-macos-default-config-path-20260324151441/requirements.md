# Requirements: macOS default config path documentation fix

## 1. Problem Statement
The repository currently documents the default config file path as `~/.config/cc-tui/profiles.toml`. Based on issue #8 and user clarification, this is incorrect for macOS. On macOS, the default config path is `~/Library/Application Support/cc-tui/profiles.toml`. At least [README.md](/home/zhengjy/workspace/cc_starter/README.md) and [CLAUDE.md](/home/zhengjy/workspace/cc_starter/CLAUDE.md) contain the wrong default path, which can mislead users configuring or troubleshooting the app on macOS.

## 2. Desired Outcome
The documentation clearly distinguishes default config paths by platform, with macOS documented as `~/Library/Application Support/cc-tui/profiles.toml` and non-macOS paths described accurately where relevant. README and CLAUDE.md use consistent wording so both external users and in-repo agents receive the same path guidance.

## 3. Constraints
- **Tech stack**: Rust CLI/TUI project; this work is documentation-only for the first version.
- **Hard boundaries**: Do not expand into implementation changes, path migration logic, or runtime detection changes. Keep the first version focused on correcting and aligning documentation in README and CLAUDE.md.

## 4. Scope
