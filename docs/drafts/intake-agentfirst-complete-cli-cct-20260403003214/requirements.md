# Requirements: Agent-First Complete CLI

## 1. Problem Statement

`cct` 目前的 CLI 入口主要面向人工终端使用，默认走 TUI，现有 `cct add` 也是逐步问答式交互。这种接口不适合 agent 稳定调用，因为调用方无法在一次命令里声明所有必要参数，也缺少稳定的机器可读输出契约。用户需要一个完整的、agent-first 的 CLI，使 agent 能显式完成 backend 选择、profile 选择、profile 新增与启动，而不依赖 TUI 或临时交互补全。

## 2. Desired Outcome

`cct` 提供一组显式子命令，至少覆盖 `list`、`add`、`launch` 三个动作，且第一版默认以机器可读 JSON 作为输出格式。调用方可以通过单条命令一次性给出所有必要参数，不进入交互、不依赖 TTY、错误和成功结果都有稳定结构。对于启动场景，调用方必须显式指定 `--backend` 和 `--profile`，从而形成清晰的 “backend selection -> profile selection” 调用链路。

## 3. Constraints
- **Tech stack**: 继续使用当前 Rust + clap + serde/toml + ratatui 架构，不引入与项目方向不符的大型新依赖；保持现有模块化边界，CLI 设计优先复用 `config`、`launch` 等现有能力。
- **Hard boundaries**:
  - 第一版必须是 agent-first，不以人工交互为默认契约。
  - 子命令体系应显式、稳定，避免把根命令继续扩展成难以预测的模式切换器。
  - `launch` 第一版必须要求同时传入 `--backend` 和 `--profile`，不能只靠 profile 隐式推断。
  - 输出契约必须默认 JSON，且 `list`、`add`、`launch` 的成功/失败结果都要可机器解析。
  - 范围至少覆盖 `list` + `add` + `launch`，形成 agent 可发现、可创建、可执行的最小闭环。
  - 现有 TUI 能力和现有 profile 配置格式应尽量保持兼容，不因新增 agent CLI 破坏当前用户路径。

## 4. Scope
### In Scope
- 新增显式子命令体系，例如 `cct list`、`cct add`、`cct launch`
- 为 agent 调用重新定义参数面，保证单条命令即可声明所有必要参数
- `list` 支持按 backend 枚举可用 profiles，并返回稳定 JSON
- `add` 支持通过显式 flags 一次性创建 profile，不进入逐步问答交互
- `launch` 支持通过 `--backend` + `--profile` 精确定位并启动 profile
- 默认 JSON 输出，以及面向成功/失败结果的统一结构化响应
- 帮助文档、错误提示、参数校验针对 agent 场景优化
- 单元测试和集成测试覆盖 `list/add/launch` 的完整命令契约
- 在不破坏现有配置文件的前提下复用当前 profile/backend 模型
- 明确 CLI 与现有 TUI/旧交互式 `add` 的关系与兼容策略

### Out of Scope
- 仅面向人工的多轮交互式补问作为第一版主路径
- 只做文本输出、不提供 JSON 契约
- 只实现 `launch` 而不提供 `list` / `add`
- 新增与 agent 调用目标无关的复杂 backend 抽象或大规模架构重写
- 超出当前 profile/backend 模型范围的远期工作流编排
- 需要联网或依赖外部服务才能完成的 CLI 基础契约定义

## 5. Acceptance Criteria
- [ ] `cct` 提供显式子命令体系，至少包含 `list`、`add`、`launch`
- [ ] `list` 能返回 backend/profile 清单，默认输出为稳定 JSON
- [ ] `add` 支持通过单条命令传入所有必要参数创建 profile，无需交互补问
- [ ] `launch` 要求显式传入 `--backend` 与 `--profile`，并据此执行目标 profile
- [ ] 三个子命令的成功结果和错误结果都具备可机器解析的 JSON 结构
- [ ] 缺失参数、无效 backend、profile 不存在、参数冲突等错误具备稳定错误码或稳定错误字段
- [ ] CLI 在非 TTY / agent 场景下不会意外降级为交互模式
- [ ] 现有 profile 配置和现有 TUI 主流程不因该 CLI 设计而回归损坏
- [ ] README / `--help` / 命令帮助文案说明新的 agent-first 调用方式
- [ ] 测试覆盖 `list`、`add`、`launch` 的命令契约与端到端主路径
