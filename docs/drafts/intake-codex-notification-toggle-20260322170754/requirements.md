# Requirements: Codex notification toggle via [n] key

## 1. Problem Statement

当前 `cct` 已支持 Codex backend 的 `full_auto` 切换，但还不能在 TUI 主界面快速控制 Codex CLI 的通知功能。用户现在只能依赖手动编辑 `$CODEX_HOME/config.toml`，无法像其他 profile 开关一样从选中的 Codex profile 直接切换通知状态。

## 2. Desired Outcome

在 Codex tab 的 Normal mode 下，按 `n` 切换当前选中 profile 的通知开关，并将该状态持久化到 `profiles.toml`。启动该 profile 时，`generate_codex_config()` 根据该 profile 生成 `$CODEX_HOME/config.toml`，写入 `[tui] notifications = <bool>`。本次仅支持内置通知布尔开关，不纳入 `notification_method` 或 `notify` 的热键/表单配置。

## 3. Constraints

- **Tech stack**: Rust, ratatui, serde, toml, toml_edit
- **Hard boundaries**:
  - 持久化源必须是 `profiles.toml`，不能只改生成后的 `$CODEX_HOME/config.toml`
  - 保持现有架构: `CODEX_HOME/config.toml` 继续由 launch 阶段从 profile 派生生成
  - 只支持 Codex backend 的 Normal mode 热键切换
  - 本次不扩展 add form，不新增 `notification_method` 或 `notify` 字段
  - 如果需要修改 `profiles.toml`，应沿用已有 toml-edit surgical edit 模式，保留注释和格式

## 4. Scope

### In Scope

- `config.rs`: 为 Codex profile 增加通知开关字段，并提供对应持久化切换能力
- `main.rs`: Codex tab 下新增 `n` 热键处理，切换当前 profile 的通知状态
- `launch.rs`: `generate_codex_config()` 从 profile 写出 `[tui] notifications = true|false`
- 为新字段和新切换行为补充测试

### Out of Scope

- `notification_method = "auto" | "osc9" | "bel"` 的配置支持
- `notify = [...]` 外部脚本通知支持
- add form / `cct add` 新增通知字段
- Claude backend 的 `n` 键行为
- 新增 toast、弹窗或其他 UI 交互模式

## 5. Acceptance Criteria

- [ ] Codex profile 有独立的通知布尔配置，持久化在 `profiles.toml`
- [ ] 在 Codex tab 的 Normal mode 按 `n`，当前 profile 的通知值取反并持久化
- [ ] 重新加载 profile 后，通知值仍可从 `profiles.toml` 正确读回
- [ ] 启动 Codex profile 时，生成的 `$CODEX_HOME/config.toml` 包含 `[tui]` 表，且 `notifications` 值与 profile 一致
- [ ] Claude backend 现有热键行为不变
- [ ] `notification_method` 和 `notify` 不会被本次实现误写入生成配置
- [ ] 相关单元测试和现有测试保持通过
