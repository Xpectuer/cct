# Requirements: Codex API Key Edit Sync

## 1. Problem Statement

When a user edits a Codex profile's API key in the TUI (via the inline edit pane), the updated key is correctly persisted to `profiles.toml` under `OPENAI_API_KEY`. However, when the user subsequently launches the Codex profile, the codex CLI still reads the old key from `auth.json`. This is because `cct`'s `exec_codex()` flow generates `config.toml` in `CODEX_HOME` but never writes or updates `auth.json`.

**GitHub Issue**: [#11 — Codex edit api key won't work](https://github.com/Xpectuer/cc_starter/issues/11)

## 2. Desired Outcome

After editing a Codex profile's API key, launching that profile causes codex CLI to use the new key. The fix ensures `auth.json` in `$CODEX_HOME` is always written with the current profile's API key at launch time, so no stale credential can persist.

## 3. Constraints
- **Tech stack**: Rust, launch.rs module only
- **Hard boundaries**:
  - auth.json format: `{"openai_api_key": "sk-..."}` in `$CODEX_HOME/auth.json`
  - Must follow existing pattern: pure builder function + thin effectful wrapper
  - Must not touch config.rs / app.rs / ui.rs
  - Must preserve KISS principle — minimal change, no over-engineering

## 4. Scope
### In Scope
- New `write_codex_auth()` function in `launch.rs`
- Call `write_codex_auth()` from `exec_codex()` after `generate_codex_config()`
- Unit test for `write_codex_auth()` verifying correct JSON output
- Integration test verifying `exec_codex` flow includes auth.json write

### Out of Scope
- Editing auth.json at profile-edit time (TUI save)
- Changes to config.rs, app.rs, or ui.rs
- Support for non-OpenAI auth providers
- Migration of existing stale auth.json files

## 5. Acceptance Criteria
- [ ] `write_codex_auth()` writes `{"openai_api_key": "<key>"}` to `$CODEX_HOME/auth.json`
- [ ] `exec_codex()` calls `write_codex_auth()` before exec-replacing with codex
- [ ] Unit test confirms auth.json content matches profile API key
- [ ] Integration test confirms full exec_codex flow writes auth.json
- [ ] `cargo test` passes
- [ ] `cargo clippy` passes
